🔧 Patch v1.7

Real TP Orders + Auto-Trail SL

Take-profit levels now fire as real limit close orders on Bitget. The stop loss trails automatically to break-even at TP1 and to TP1 price after TP2. This patch is cumulative — works from any previous version.

Released 8 May 2026  ·  Works from any previous version (v1.0–v1.6)

What Changed

This patch is cumulative. Everything from v1.5 and v1.6 is included — if you're coming from an older install you only need this one page.

New in v1.7
🎯
Improvement

Take-profit levels now place real limit close orders on Bitget

Previously TPs were tracked internally in simulation — the bot would mark a TP as hit but never actually place an order to close the position on Bitget. v1.7 places a real limit sell (or buy for shorts) order at each TP price the moment a position opens. Bitget executes the close automatically when the price is reached.

🔍
Improvement

TP fill monitoring — checks order status every cycle

Each bot cycle checks whether outstanding TP limit orders have been filled. When TP1 fills, the bot records the partial close and advances the position state. This keeps the internal tracker in sync with what Bitget has actually executed.

🛡️
Improvement

Stop loss trails automatically after each TP hit

After TP1 fills: SL moves to entry price (break-even — the trade cannot lose money from this point). After TP2 fills: SL moves to TP1 price (locking in partial profit on the remaining position). The bot updates the stop loss order on Bitget via the modify-order endpoint.

Included from v1.6
🚨
Critical Fix

Ghost order prevention — failed orders no longer block daily limit

Failed API orders were being logged to the CSV with orderId: unknown, counting against the daily trade limit. Fix: orders are only logged when a real orderId is returned by Bitget.

⚡
Improvement

Leverage set inline — no separate API call, no permission error

Leverage is now passed directly in the place-order body rather than via a separate set-leverage call that requires an extra API permission (error 40014).

🔧
Fix

Margin mode on close orders — prevents error 400172

marginMode is now always specified on close orders to match how the position was originally opened (isolated). Omitting it caused error 400172; using the wrong mode caused error 22002.

Included from v1.5
🚨
Critical Fix

Bitget API bugs fixed — orders now actually fire

Three silent API failures blocked all live orders on v1.0–1.4: wrong endpoint casing (placeOrder → place-order), wrong side format for one-way mode, and wrong margin mode. Watchlist also expanded from 10 to 18 pairs.

Code Reference

Key changes in bot-ironclad.js for transparency.

Real TP limit close orders at position open

// Before (v1.6) — TPs tracked in simulation only, no real orders placed:
addOpenPosition({ id: orderId, tp1, tp2, tp3, tp1Hit: false, ... });

// After (v1.7) — real limit close orders placed immediately:
addOpenPosition({ id: orderId, tp1, tp2, tp3, tp1Hit: false, ... });

async function placeTpOrder(symbol, closeSide, qty, tpPrice) {
  return await bitgetRequest('POST', '/api/v2/mix/order/place-order', {
    symbol, productType: 'USDT-FUTURES', marginCoin: 'USDT',
    marginMode: 'isolated',
    size: qty.toString(),
    side: closeSide,       // 'sell' for long, 'buy' for short
    tradeSide: 'close',
    orderType: 'limit',
    price: tpPrice.toFixed(4),
    leverage: CONFIG.leverage.toString(),
  });
}

TP fill monitoring — run each cycle

// After (v1.7) — check TP order status each bot cycle:
async function checkTpFills(pos) {
  if (!pos.tp1OrderId || pos.tp1Hit) return;
  const r = await bitgetRequest('GET',
    `/api/v2/mix/order/detail?symbol=${pos.symbol}&orderId=${pos.tp1OrderId}`, null);
  if (r?.data?.state === 'filled') {
    pos.tp1Hit = true;
    pos.currentSL = pos.entry;   // trail SL to break-even
    console.log(`  ✅ ${pos.symbol} TP1 filled — SL moved to break-even`);
  }
}

Auto-trail SL after each TP

// After (v1.7) — SL trails automatically to lock in profit:
if (pos.tp1Hit && !pos.tp2Hit && tp2Filled) {
  pos.tp2Hit = true;
  pos.currentSL = pos.tp1;  // lock in TP1 profit on remaining position
  await modifyStopLoss(pos.symbol, pos.currentSL);
}
if (!pos.tp1Hit && tp1Filled) {
  pos.tp1Hit = true;
  pos.currentSL = pos.entry; // break-even — cannot lose
  await modifyStopLoss(pos.symbol, pos.currentSL);
}
