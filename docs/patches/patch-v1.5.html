🔧 Patch v1.5

Bitget API Fix + 18-Pair Watchlist

Critical order placement fixes discovered during go-live testing. If you deployed v1.0–1.4 your bot has never placed a real order — this patch fixes that.

Released 6 May 2026  ·  Affects all installs on v1.0–1.4

What Changed

Three critical API bugs prevented any live orders from firing. If you were in paper mode you won't have noticed — all trade decisions were logged correctly, only the order submission was silently failing.

🚨
Critical Fix

Wrong API endpoint format — no orders were ever placed

Bitget v2 endpoints use kebab-case. placeOrder returns 404. The correct endpoint is place-order. This affected both bot-ironclad.js and bot-hype-manager.js.

🚨
Critical Fix

Wrong margin mode — orders rejected with error 400172

marginMode: "crossed" is not valid for isolated futures accounts. Must be marginMode: "isolated".

🚨
Critical Fix

Wrong side format for one-way position mode — error 40774

Bitget one-way mode requires side: "buy"/"sell" + tradeSide: "open". The old format side: "open_long" is hedge-mode only and causes 40774 errors.

🛡️
Improvement

set-leverage wrapped in try/catch — no longer fatal

The set-leverage API call requires an account management permission (error 40014) not included in standard "Futures Order" API keys. The bot now catches this gracefully and continues — use manually set leverage on BitGet instead.

Watchlist

Expanded from 10 → 18 pairs

Added 8 new pairs across DePIN/AI, privacy, BlockDAG and independent L1 narratives. All CoinGecko top 100. New pairs highlighted below.

How to Apply This Patch

Four things to do — estimated time 10 minutes.

Set BitGet to one-way position mode

In BitGet → Futures → Settings ⚙️ → Position Mode → select One-Way.

If you skip this every order will fail with error 40774.

Pull the latest code from GitHub

In your local repo folder:

git pull origin main

This brings in the fixed bot-ironclad.js and bot-hype-manager.js. The changes are already committed — you don't need to edit any files manually.

✅ The three API bugs are fixed in the repo. git pull is all you need for the code.

Redeploy to Railway

Railway auto-deploys when you push to main. If you just pulled (not pushed) you'll need to trigger a redeploy:

git commit --allow-empty -m "Trigger redeploy — patch v1.5" && git push origin main

Or use the Railway dashboard → your service → Redeploy.

Confirm IRONCLAD_PAPER is set correctly in Railway

Railway → your service → Variables. Check IRONCLAD_PAPER:

IRONCLAD_PAPER=          ← blank → still in paper mode (bad)
IRONCLAD_PAPER=0         ← zero → still in paper mode (bad)
IRONCLAD_PAPER=false     ← exact string → live trading ✅
⚠️ The value must be the string false. Any other value — including blank — defaults the bot to paper mode and no real orders are placed.

Exact Code Changes (for reference)

If you made manual edits to the bot code and can't simply pull, apply these changes manually.

bot-ironclad.js — placeOrder()

// BEFORE (broken)
return bitgetRequest('POST', '/api/v2/mix/order/placeOrder', {
  symbol, productType: 'USDT-FUTURES', marginCoin: 'USDT',
  marginMode: 'crossed',
  side: side === 'long' ? 'open_long' : 'open_short',
  orderType: 'market', size: quantity.toString(), ...
});

// AFTER (fixed)
return bitgetRequest('POST', '/api/v2/mix/order/place-order', {
  symbol, productType: 'USDT-FUTURES', marginCoin: 'USDT',
  marginMode: 'isolated',
  side: side === 'long' ? 'buy' : 'sell',
  tradeSide: 'open',
  orderType: 'market', size: quantity.toString(), ...
});

bot-hype-manager.js — placeCloseLimit()

// BEFORE (broken)
return bitgetRequest('POST', '/api/v2/mix/order/placeOrder', {
  symbol, productType, marginCoin,
  side: 'close_long',
  orderType: 'limit', price: price.toString(), size: qty.toString(),
});

// AFTER (fixed)
return bitgetRequest('POST', '/api/v2/mix/order/place-order', {
  symbol, productType, marginCoin,
  marginMode: 'isolated',
  side: 'sell',
  tradeSide: 'close',
  orderType: 'limit', price: price.toString(), size: qty.toString(),
});
