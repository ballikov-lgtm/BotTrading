🔧 Patch v1.6

Ghost Trades Fix + HYPE Margin Mode

Two bugs discovered during live testing. Failed order attempts were filling the daily trade limit, and the HYPE TP manager couldn't place close orders due to the wrong margin mode.

Released 7 May 2026  ·  Requires v1.5 — apply in order

What Changed

Both issues were invisible in paper trading — they only showed up once the bot was running live against the real API.

🚨
Critical Fix

Failed orders were logged to CSV — blocking future signals

When a live order failed (API error, wrong endpoint, bad params), the bot still wrote it to trades-ironclad.csv with orderId: unknown. Since the daily trade limit counts CSV entries per symbol, these ghost entries ate up the limit — so after a few failed cycles, every symbol showed "Daily limit reached" even though no real trades had opened. Fix: orders are now only logged to the CSV and open-positions tracker when a real orderId is returned by the API.

🚨
Critical Fix

HYPE TP manager: error 400172 "margin mode cannot be empty"

After the v1.5 fix removed marginMode from close orders (which caused error 22002), Bitget started returning 400172 — the field is required but must match the position's actual margin mode. The HYPE position was opened in crossed margin mode (manual open), so close orders must also specify marginMode: "crossed". The bot was cycling every 15 min without ever placing the TP1 order.

🛡️
Improvement

Leverage passed inline in order body — no separate API call needed

The separate set-leverage API call requires an "Account Management" permission (error 40014) that standard futures API keys don't have. This caused new positions to open at whatever leverage was already set on the account (e.g. 10×) rather than the configured 3×. Fix: leverage is now passed directly in the place-order body, which sets it at order time without needing the extra permission.

How to Apply This Patch

Two steps — estimated time 5 minutes.

Pull the latest code from GitHub

In your local repo folder:

git pull origin main

This brings in the fixed bot-ironclad.js (ghost order fix + inline leverage) and bot-hype-manager.js (crossed margin mode). No manual file edits needed.

✅ All three fixes are already committed to the repo. git pull is all you need for the code.

Redeploy to Railway

Railway auto-deploys when you push to main. If you just pulled (not pushed), trigger a redeploy:

git commit --allow-empty -m "Trigger redeploy — patch v1.6" && git push origin main

Or use the Railway dashboard → your service → Redeploy.

⚠️ If you have a HYPE position open: check what margin mode it was opened in (BitGet → Futures → Positions → tap the position → margin mode shown). If it was isolated rather than crossed, you'll need to change marginMode: 'crossed' to marginMode: 'isolated' in bot-hype-manager.js line ~110 before deploying.

Code Reference

For transparency — exactly what changed in each file.

bot-ironclad.js — Ghost order prevention

After placeOrder() returns, check for a real orderId before logging:

// Before (v1.5) — always logged regardless of success:
appendTrade(row);
addOpenPosition({ id: order.orderId || 'unknown', ... });

// After (v1.6) — only log when order actually landed:
const realOrderId = order?.orderId || order?.data?.orderId;
if (!realOrderId && !CONFIG.paperTrading) {
  console.log('  ✗ Order failed — NOT logging (no orderId).');
  continue;
}
appendTrade(row);
addOpenPosition({ ... });

bot-hype-manager.js — Correct margin mode on close orders

// Before (v1.5) — marginMode omitted → error 400172:
const r = await bitgetRequest('POST', '/api/v2/mix/order/place-order', {
  symbol, productType, marginCoin,
  side: 'sell', tradeSide: 'close', ...
});

// After (v1.6) — marginMode matches position (crossed):
const r = await bitgetRequest('POST', '/api/v2/mix/order/place-order', {
  symbol, productType, marginCoin,
  marginMode: 'crossed',
  side: 'sell', tradeSide: 'close', ...
});

bot-ironclad.js — Inline leverage on open orders

// Before (v1.5) — relied on separate set-leverage call (fails silently):
await setLeverage(symbol, CONFIG.leverage); // error 40014 — no effect
const body = { symbol, side, tradeSide: 'open', ... };

// After (v1.6) — leverage passed directly in order body:
const body = {
  symbol, side, tradeSide: 'open',
  leverage: CONFIG.leverage.toString(), // set at order time, no permission needed
  ...
};
